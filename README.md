Тисин Эмиль 9-3-по-22 
Цель приложения: Помочь студентам и швольникам эффективно планировать учебу,не забывать о дедлайнах и поддерживать мотивацию спомощью умных напоминаний и аналитики прогресса.
Проблема
	•	Пользователи забывают о дедлайнах и контрольных
	•	Учебные задания хранятся в разных местах (тетради, чаты, сайты)
	•	Нет понимания, куда уходит время и почему падает продуктивность
	•	Обычные планировщики не адаптированы под учебный процесс
Решение:
StudyFlow объединяет задания, расписание и напоминания в одном приложении и помогает выстроить понятный учебный процесс.
Целевая аудитория

Целевая аудитория
	•	Школьники 8–11 классов
	•	Студенты колледжей и вузов
	•	Онлайн-студенты и слушатели курсов

 Особенности аудитории
	•	Активно используют смартфоны
	•	Часто перегружены задачами
	•	Нуждаются в простом и наглядном интерфейсе
 
  Основной функционал приложения
  Основные функции
	•	Создание учебных задач и дедлайнов
	•	Календарь занятий и экзаменов
	•	Разделение задач по предметам
	•	Отслеживание прогресса и статистики
	•	Синхронизация между устройствами

 Дополнительно
	•	Цветовая маркировка предметов
	•	Режим «Подготовка к экзамену»
  Используемые API

 Используемые API и их назначение
	•	Firebase Authentication API
→ регистрация и вход пользователей
	•	Firebase Firestore / Realtime Database
→ хранение задач, расписаний и прогресса
	•	Calendar API
→ синхронизация с календарём устройства
	•	Analytics API
→ анализ активности и вовлечённости пользователей
Push-уведомления: роль и сценарии

 Роль push-уведомлений
	•	Помогают не пропускать важные события
	•	Повышают дисциплину и мотивацию
	•	Удерживают пользователя в приложении

 Сценарии использования
	•	Напоминание о дедлайне за 1 день и за 1 час
	•	Уведомление о начале занятия
	•	Мотивационные сообщения («Вы закрыли все задачи на сегодня!»)
	•	Напоминание о длительном отсутствии активности
